1. Wprowadzenie

Edward Cullen - Personal Organizer to kompleksowa aplikacja webowa służąca do zarządzania codziennymi zadaniami, nawykami, finansami oraz życzeniami. Aplikacja została zaprojektowana jako Single Page Application (SPA) wykorzystująca wyłącznie technologie frontendowe, bez konieczności użycia frameworków zewnętrznych.

Główne założenia projektu:
  • Aplikacja działa całkowicie po stronie klienta (client-side)
  • Dane przechowywane są lokalnie w przeglądarce (LocalStorage)
  • Brak zależności zewnętrznych - czysty JavaScript, HTML i CSS
  • Responsywny design dostosowany do różnych rozmiarów ekranów
  • Intuicyjny interfejs użytkownika z ciemnym motywem

2. Przegląd projektu

Aplikacja składa się z pięciu głównych modułów funkcjonalnych:

Moduły aplikacji:

  1. Habit Tracker (HT) - śledzenie codziennych nawyków z wizualizacją kalendarzową
  2. Kosztorys (KT) - zarządzanie finansami (wydatki i przychody) z analityką
  3. TodoList (TD) - lista zadań z priorytetami i kategoriami
  4. Calendar View (CV) - widok kalendarzowy z integracją zadań
  5. WishList (WL) - lista życzeń z możliwością śledzenia kosztów

5. Architektura aplikacji

5.1. Wzorzec architektoniczny

Aplikacja wykorzystuje wzorzec Modular Architecture z centralnym zarządzaniem stanem. Każdy moduł jest niezależny, ale współdzieli wspólny obiekt stanu.

5.2. Zarządzanie stanem

Wszystkie dane aplikacji przechowywane są w centralnym obiekcie state:

const state = {
    currentModule: 'habits',           // Aktualnie aktywny moduł
    habits: [],                        // Tablica nawyków
    transactions: [],                   // Tablica transakcji
    todos: [],                         // Tablica zadań
    wishes: [],                        // Tablica życzeń
    currentMonth: number,              // Aktualny miesiąc (0-11)
    currentYear: number                // Aktualny rok
};

5.3. Przepływ danych

  1. Inicjalizacja - dane ładowane z LocalStorage do obiektu state
  2. Interakcja użytkownika - zdarzenia wywołują funkcje modyfikujące state
  3. Renderowanie - funkcje renderujące aktualizują DOM na podstawie state
  4. Zapis - zmiany w state są zapisywane do LocalStorage

5.4. Struktura modułów

Każdy moduł posiada następującą strukturę funkcji:

  • init[Module]Module() - funkcja inicjalizująca moduł
  • render[Module]() - funkcja renderująca dane modułu
  • Funkcje pomocnicze specyficzne dla modułu

6. Funkcje pomocnicze

6.1. Funkcje utility

6.1.1. saveToStorage(key, data)

Opis: Zapisuje dane do LocalStorage w formacie JSON.

Parametry:

  • key - klucz pod którym dane są zapisywane
  • data - dane do zapisania (obiekt lub tablica)
function saveToStorage(key, data) {
    localStorage.setItem(key, JSON.stringify(data));
}

6.1.2. formatDate(date)

Opis: Formatuje datę do polskiego formatu (DD.MM.YYYY).

Parametry:

  • date - obiekt Date lub string ISO
function formatDate(date) {
    return new Date(date).toLocaleDateString('pl-PL');
}

6.1.3. getDaysInMonth(month, year)

Opis: Zwraca liczbę dni w danym miesiącu.

Parametry:

  • month - numer miesiąca (0-11)
  • year - rok
function getDaysInMonth(month, year) {
    return new Date(year, month + 1, 0).getDate();
}

6.1.4. getFirstDayOfMonth(month, year)

Opis: Zwraca dzień tygodnia pierwszego dnia miesiąca (0 = niedziela, 6 = sobota).

Parametry:

  • month - numer miesiąca (0-11)
  • year - rok
function getFirstDayOfMonth(month, year) {
    return new Date(year, month, 1).getDay();
}

6.2. Funkcje nawigacji

6.2.1. initNavigation()

Opis: Inicjalizuje obsługę kliknięć na elementy nawigacji w sidebarze.

6.2.2. switchModule(module)

Opis: Przełącza aktywny moduł aplikacji.

Parametry:

  • module - identyfikator modułu ('habits', 'budget', 'todo', 'calendar', 'wishlist')

Działanie:

  1. Aktualizuje aktywny element w nawigacji
  2. Ukrywa wszystkie moduły
  3. Pokazuje wybrany moduł
  4. Aktualizuje tytuł w topbarze
  5. Inicjalizuje wybrany moduł

7. Przechowywanie danych

7.1. LocalStorage API

Aplikacja wykorzystuje Web Storage API (LocalStorage) do przechowywania wszystkich danych po stronie klienta. Dane są zapisywane w formacie JSON.

7.2. Klucze LocalStorage

Klucz Typ danych Opis
'habits' Array Tablica obiektów nawyków
'transactions' Array Tablica obiektów transakcji
'todos' Array Tablica obiektów zadań
'wishes' Array Tablica obiektów życzeń

7.3. Format danych

Wszystkie dane są serializowane do JSON przed zapisem i deserializowane po odczytaniu:

// Zapis
localStorage.setItem('habits', JSON.stringify(state.habits));

// Odczyt
state.habits = JSON.parse(localStorage.getItem('habits') || '[]');
Uwaga: LocalStorage ma limit około 5-10 MB na domenę. Dla większości przypadków użycia aplikacji to wystarczająca ilość miejsca. W przypadku przekroczenia limitu, aplikacja może nie zapisywać nowych danych.

7.4. Persystencja danych

Dane przechowywane w LocalStorage są:

  • Trwałe - pozostają po zamknięciu przeglądarki
  • Specyficzne dla domeny - dostępne tylko dla tej samej domeny
  • Synchroniczne - operacje są synchroniczne (blokujące)

9. Instalacja i uruchomienie

9.1. Wymagania

  • Nowoczesna przeglądarka internetowa (Chrome, Firefox, Edge, Safari)
  • Wsparcie dla JavaScript ES6+
  • Wsparcie dla LocalStorage API
  • Wsparcie dla Canvas API (dla wykresów)

9.2. Instalacja

Aplikacja nie wymaga instalacji. Wystarczy:

  1. Pobrać pliki projektu
  2. Upewnić się, że struktura folderów jest zachowana: 
    Edward-Cullen/
├── Edward.html
├── script.js
└── style/
    └── style.css

9.3. Uruchomienie

Aplikację można uruchomić na dwa sposoby:

9.3.1. Lokalnie (file://)

Otwórz plik Edward.html bezpośrednio w przeglądarce. Niektóre funkcje mogą być ograniczone z powodu polityki CORS.

9.3.2. Przez serwer lokalny (zalecane)

Uruchom lokalny serwer HTTP. Przykłady:

Python 3:

python -m http.server 8000

Następnie otwórz: http://localhost:8000/Edward.html

Node.js (http-server):

npx http-server

PHP:

php -S localhost:8000
Zalecenie: Używanie lokalnego serwera HTTP jest zalecane, ponieważ zapewnia pełną funkcjonalność aplikacji i unika problemów z CORS.

10. Instrukcja użytkowania

10.1. Pierwsze uruchomienie

  1. Otwórz aplikację w przeglądarce
  2. Domyślnie otworzy się moduł Habit Tracker
  3. Użyj nawigacji w lewym sidebarze, aby przełączać się między modułami

10.2. Habit Tracker

  1. Kliknij przycisk "+ Dodaj nawyk"
  2. Wpisz nazwę nawyku i kliknij "Zapisz"
  3. Klikaj na dni w kalendarzu, aby oznaczyć je jako wykonane (zielone)
  4. Dni w przeszłości, które nie zostały wykonane, automatycznie stają się czerwone
  5. Sprawdzaj postęp na pasku postępu pod kalendarzem

10.3. Kosztorys

  1. Kliknij "+ Wydatek" lub "+ Przychód"
  2. Wypełnij formularz: kwota, opis, kategoria
  3. Kliknij "Zapisz"
  4. Sprawdzaj podsumowanie finansowe na górze
  5. Przeglądaj wykres kołowy wydatków według kategorii
  6. Przeglądaj listę wszystkich transakcji poniżej

10.4. TodoList

  1. Kliknij "+ Dodaj zadanie"
  2. Wypełnij formularz: tytuł, opis (opcjonalnie), priorytet, kategorię, datę (opcjonalnie)
  3. Kliknij "Zapisz"
  4. Użyj filtrów, aby filtrować zadania po priorytecie, kategorii lub statusie
  5. Kliknij "Sortuj", aby posortować zadania według priorytetu
  6. Zaznacz checkbox, aby oznaczyć zadanie jako wykonane

10.5. Calendar View

  1. Użyj przycisków "‹" i "›", aby nawigować między miesiącami
  2. Dni z zadaniami wyświetlają liczbę zadań
  3. Kliknij na dzień, aby zobaczyć listę zadań na ten dzień
  4. Dzisiejszy dzień jest wyróżniony niebieską ramką

10.6. WishList

  1. Kliknij "+ Dodaj życzenie"
  2. Wypełnij formularz: nazwa, cena (opcjonalnie), opis (opcjonalnie)
  3. Kliknij "Zapisz"
  4. Zaznacz checkbox, aby oznaczyć życzenie jako spełnione
  5. Sprawdzaj podsumowanie statystyk w panelu po prawej stronie

10.7. Zarządzanie danymi

Uwaga: Wszystkie dane są przechowywane lokalnie w przeglądarce. Wyczyszczenie danych przeglądarki (cache, cookies, LocalStorage) spowoduje utratę wszystkich danych aplikacji.

10.8. Porady

  • Regularnie sprawdzaj postęp w nawykach, aby utrzymać motywację
  • Kategoryzuj transakcje, aby lepiej zrozumieć swoje wydatki
  • Używaj priorytetów w zadaniach, aby skupić się na najważniejszych rzeczach
  • Przypisuj daty do zadań, aby lepiej je planować w kalendarzu
  • Śledź ceny życzeń, aby móc planować budżet
